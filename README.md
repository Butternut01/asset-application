# 📦 Система учета оборудования (Asset Application)

Современное веб-приложение для инвентаризации и отслеживания активов организации с поддержкой QR-кодов и аналитической панелью управления.

---

## ✨ Основные возможности

*   **📊 Аналитическая панель (Dashboard):** Визуализация данных об активах, их стоимости и распределении.
*   **📂 Управление инвентарем:** Полный цикл CRUD (создание, чтение, обновление, удаление) для оборудования.
*   **🔍 Поиск и фильтрация:** Быстрый поиск по названию, серийному номеру или ответственному лицу.
*   **📱 QR-коды:** Автоматическая генерация QR-кодов для каждого актива. При сканировании открывается страница с публичной информацией об объекте.
*   **🔐 Безопасность:** Защищенная зона администратора с авторизацией (Passport.js).
*   **📥 Импорт/Экспорт:** Поддержка работы с данными в формате JSON/Excel (через API).

---

## 🛠 Технологический стек

*   **Runtime:** Node.js
*   **Framework:** Express.js
*   **Database:** MongoDB (Mongoose ODM)
*   **Frontend:** EJS (Embedded JavaScript templates)
*   **Auth:** Passport.js (Local Strategy)
*   **Styling:** Vanilla CSS + современные UI паттерны

## 🚀 Быстрый старт

### 1. Предварительные требования

Убедитесь, что у вас установлены:
*   [Node.js](https://nodejs.org/) (версия 14 или выше)
*   [MongoDB](https://www.mongodb.com/try/download/community) (локально или облачная база MongoDB Atlas)

### 2. Клонирование и установка

```bash
# Перейдите в папку проекта
cd asset-application

# Установите зависимости
npm install
```

### 3. Настройка окружения

Создайте файл `.env` в корневой директории и добавьте в него следующие параметры:

```env
MONGODB_URI=mongodb://localhost:27017/inventoryDB
PORT=3001
SESSION_SECRET=your_secret_key_here
```

### 4. Создание учетной записи администратора

Для первого входа в систему необходимо создать пользователя-администратора:

```bash
node createAdmin.js
```
*По умолчанию будет создан пользователь: Логин: `admin`, Пароль: `admin123`*

### 5. Запуск приложения

**Режим разработки (с автоперезагрузкой):**
```bash
npm run dev
```

**Обычный запуск:**
```bash
npm start
```

Приложение будет доступно по адресу: `http://localhost:3001`

## 📂 Структура проекта

*   `app.js` — точка входа в приложение и конфигурация сервера.
*   `routes/` — маршруты API и страниц (инвентарь, админка).
*   `models/` — схемы данных Mongoose (Inventory, User).
*   `views/` — шаблоны страниц EJS.
*   `public/` — статические файлы (CSS, клиентский JS, изображения).
*   `middleware/` — посредники для проверки аутентификации.

---

## 📖 Использование

1.  **Вход:** Перейдите на `/login` для входа в панель управления.
2.  **Добавление:** В панели администратора нажмите "Добавить актив" для внесения нового оборудования.
3.  **QR-печать:** Для каждого актива доступна кнопка печати QR-кода. Наклейте его на оборудование.
4.  **Просмотр:** Любой сотрудник может сосканировать QR-код телефоном и увидеть информацию об активе без необходимости логина.
